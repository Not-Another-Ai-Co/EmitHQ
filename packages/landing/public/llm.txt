# EmitHQ — Webhook Infrastructure Platform

> Open-source webhook infrastructure for SaaS teams. Send and receive webhooks with Standard Webhooks signing, automatic retries, and a real-time dashboard. From $49/mo.

## Quick Start (API-only, no browser required)

1. Create an account:
   curl -X POST https://api.emithq.com/api/v1/signup \
     -H "Content-Type: application/json" \
     -d '{"email": "you@example.com", "password": "your-password", "orgName": "Your Company"}'
   # Returns: { data: { orgId, apiKey, userId, tier, eventLimit } }

2. Create an application:
   curl -X POST https://api.emithq.com/api/v1/app \
     -H "Authorization: Bearer emhq_YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{"name": "my-app"}'

3. Create an endpoint (where webhooks will be delivered):
   curl -X POST https://api.emithq.com/api/v1/app/APP_ID/endpoint \
     -H "Authorization: Bearer emhq_YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{"url": "https://your-server.com/webhooks"}'
   # Returns signing secret (whsec_...) — save it for verification

4. Send a webhook event:
   curl -X POST https://api.emithq.com/api/v1/app/APP_ID/msg \
     -H "Authorization: Bearer emhq_YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{"eventType": "invoice.paid", "payload": {"amount": 9900}}'

5. Check delivery status:
   curl https://api.emithq.com/api/v1/app/APP_ID/msg/MSG_ID \
     -H "Authorization: Bearer emhq_YOUR_API_KEY"

## Authentication

All API requests use Bearer token authentication:
  Authorization: Bearer emhq_YOUR_API_KEY

API keys are created during signup (POST /api/v1/signup) or via the dashboard.
Keys use the `emhq_` prefix. Store securely — plaintext is shown once.

## Key Concepts

- **Application**: A logical grouping (usually one per customer of yours)
- **Endpoint**: A URL where webhooks are delivered, with its own signing secret
- **Message**: A webhook event sent to all matching endpoints
- **Delivery Attempt**: One HTTP POST to one endpoint for one message
- **Signing**: Standard Webhooks spec (HMAC-SHA256). Headers: webhook-id, webhook-timestamp, webhook-signature

## Retry Strategy

Failed deliveries retry with exponential backoff + jitter:
Attempt 1: immediate, 2: ~30s, 3: ~2m, 4: ~15m, 5: ~1h, 6: ~4h, 7: ~12h, 8: ~12h
Non-retriable: 400, 401, 403, 404, 410
After all retries exhausted: moved to dead-letter queue

## Quota Headers

Every authenticated response includes:
  X-EmitHQ-Quota-Limit: 100000
  X-EmitHQ-Quota-Used: 50000
  X-EmitHQ-Quota-Remaining: 50000
  X-EmitHQ-Quota-Reset: 2026-04-01T00:00:00.000Z
  X-EmitHQ-Tier: free

At 80% usage: X-EmitHQ-Quota-Warning: approaching_limit
At 95% usage: X-EmitHQ-Quota-Warning: critical_limit

When quota is exceeded (429), the response includes upgrade options:
  { error: { code: "quota_exceeded", quota: {...}, action: { type: "upgrade", tiers: [...] } } }
